OIDC / SSO Configuration

Configure OpenID Connect for single sign-on (e.g. Authentik, Keycloak). Changes take effect on the next restart.

OIDC is currently active. A “Login with OIDC” link appears on the login page.

Authentik: Admin → Providers → [provider], copy the OpenID Configuration URL and remove the trailing /.well-known/openid-configuration.

Leave blank to keep the currently stored secret unchanged.

Must exactly match the redirect URI in your provider. The callback path is always /auth/oidc/callback.

Authentik Setup Guide
1
Create an OAuth2/OIDC Provider
Admin Interface → Applications → Providers → Create → OAuth2/OpenID Provider
  • Client type: Confidential
  • Authorization flow: default-authorization-flow
  • Redirect URI:
  • Scopes: openid profile email
Copy the generated Client ID and Client Secret into the form above.
2
Create an Application
Applications → Applications → Create
Give it a name (e.g. Synergia Admin) and bind it to the provider from step 1.
3
Configure Bindings (access control)
Under the Application → Policy / Group / User Bindings tab:
  • Create a group synergia-admins in Authentik and add the admin users to it
  • Add a Group Binding → select synergia-admins — only members can log in
  • Optionally add a Policy Binding to enforce MFA on this application
4
Find the Provider (Issuer) URL
Admin → Providers → [your provider]
Copy the OpenID Configuration URL, then remove the trailing /.well-known/openid-configuration.
Example: https://auth.example.com/application/o/synergia/
5
Save and restart
Fill in the form above and click Save, then restart the manager. The login page will show a Login with OIDC link once OIDC is active.
