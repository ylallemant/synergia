Data Collection Consent Required

To participate in the Synergia cluster, you must accept the collection and centralisation of:

  • Fingerprint — a unique cryptographic identity for this worker
  • Hardware statistics — OS + version, GPU + VRAM + driver/version, CPU + core count, RAM
  • GPU availability average — a rolling mean of idle GPU load (bottom-85th-percentile, peaks excluded) sent once per minute; no load curves, no timestamps, no activity inference
  • Configuration preferences — model size limits, quantisation, preferred role
  • Automatic updates — the client will download backend binaries (e.g. llama-server) and self-update when instructed by the cluster manager

This data is tied to your worker fingerprint and used by the cluster manager to schedule work efficiently. No personal information is collected.
